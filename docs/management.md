# Project Management

# Вступ

Керування проектом є невід’ємною частиною розробки програмного забезпечення. Так, даний етап не є ключовим для всього процесу, але без його якісного втілення, успіх проекту майже неможливий.
На цьому етапі потрібно вирішити наступні питання:

* з якими проблемами доведеться стикнутися в процесі та як їх вирішувати;
* як відбуватиметься безпосердній процес розробки та як його контролювати;
* як відбуватиметься злагоджена комунікація всередині команди.

Детальний аналіз. В пропозалі був загальний аналіз
Продумати всі ризики. Шо з ними робити, як реагувати.
Є готові рішення у випадку не стандартних ситуацій.

Імовірність виникнення може бути: незначна, низька, середня, висока й дуже висока.
Ефект від впливу може бути: катастрофічний, серйозний, прийнятний або незначний.

# Засоби моніторингу

## Внутрішня комунікація

* Telegram чат для термінового вірішення питань проекту.
* GitHub issues - основне джерело інформації проекту.
* Щотижневі збір команди для синхронізації розробки ситеми, обговорень виконаної роботи, можливої зміни приорітетів задач;
* GoogleDrive - сервіс для зберігання та обміну внутрішніх файлів проекту

## Спостереження та контролювання прогресу розробки

* GitHub dashboard - задачи в очереди, задачи в процессе, выполненные задачи и требующие обсуждения на текущем этапе разработки

## Комунікація з замовником

* Відео конференція та чат через зручний для замовника сервіс
* Зустрічи з замоником
